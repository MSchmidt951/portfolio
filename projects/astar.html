A-Star Visualiser

Description

A program that visualises my implementation of the A-star pathfinding algorithm in Python.
The user can chose the start and target point. Then the algorithm finds an efficient path between those two points then the path is shown to the user as the algorithm works.
When a path has been found it will be displayed on the screen as a red line going from the starting node (blue) to the target node (pink).

Closer Look

Algorithm Explanation

The screen is split into a grid of nodes. Each node has a position which correlates to a section of the grid.
The cost is the penalty which is the sum two items: the distance to the target node and the length of the path from the start node.
They also store the previous node in the path.

To start the algorithm, a StartNode object is created at the starting position. The only difference from a normal node is that it does not have a previous node.
This is then set as the current node.

Then the current node calls the function getNeighbours.
This function checks each adjacent square in the grid.
To check if the neighbour is valid it checks:
• It is not a wall
• It is not discovered
• It is within the grid
The neighbours are added to the discovered nodes and the current nodes are moved from the discovered nodes to the visited nodes.
The code compares the cost of the discovered nodes and chooses the node with the lowest cost to check next.
This process is repeated until the target node has been found.

When the target node has been found, the path is traced back by looking recursively at each node's previous node until the starting node is reached. This is the path that the algorithm has found, displayed in red.

Obstacle Avoidance

Walls are obstacles that the algorithm cannot go through. They are stored as a list of coordinates which correlate to the grid.
I decided to include them in the program so the path will not be a straight line, providing different scenarios for the algorithm.
I used random Perlin noise which allows different structures of walls to be generated without the user having to come up with their own paths.
I used Perlin noise rather than normal noise as this groups wall sections together creating a maze-like pattern.

Adjusting Settings

I added user-friendly interaction with the program to easily use and adjust the algorithm and what environment it is in while the program is running, rather than having to stop the program, edit variables and launch the program again.
For example the user can add and delete walls manually, change the seed used to generate the walls, how many walls are generated and change the behaviour of the algorithm.
