Drone

Description

This page (and project) is not finished but shows most of what I have done.

The aim of this project was to design and build a quadcopter drone where I would use as few pre-made components as possible. This included creating the software and circuit board of the flight controller, a frame which can be made with little to no tools and choosing pre-made components (e.g. motors & propellers) which work well together for optimal performance.
I used an iterative approach to the development. This allowed me to test different ideas to gain a greater understanding of how different parameters affected performance in a real world environment. This approach enabled me to be flexible in the design to get the best optimisation possible.

Skills

When I started this project, I knew very little about how drones worked and I enjoyed learning about different components and how they work together.

Closer look

The drone flies by spinning four propellers with brushless motors.
The flight controller uses sensors to get the state of the drone (eg rotation) then calculates what speeds the induvidual motors need to spin at in order get to the state it wants to be in.
Then the flight controller sends a signal to the electronic speed controllers (ESCs) to set the motor speed.

Creating the project

Each section of the project was developed in parallel so were improved to both better suit its own function and to help improve other sections.
This means each section has its own version number and may not line up with other versions.

    An overview of the development process:

  • After deciding the initial idea of the project I ordered the cheapest ESC, propeller and motor package deal I could find to use for the initial development and so I would not waste money on getting the wrong equipment. It also would not matter if this cheap equipment got damaged
  • I then tested parts induvidually to get the design of the first prototype
  • Making and testing first prototype (to find out what is needed for a proper drone prototype & sort out any major issues)
  • Test series 2: tune values for performance and see what affects how the drone flies
  • Roll jig test: see how the drone handles long duration flight, attempting to tune 1 axis at a time
  • Test series 3: improve reliability and performance to prepare for first fully functional prototype
  • Test series 4: create a fully functioning prototype
