Tank Colosseum

Description

Tank Colosseum is a top down 2D game where 2-4 players battle each other in tanks. Inspired by Tank Trouble.

The game is played in rounds where the last tank alive wins.
Each round the tanks will spawn in random locations on a random map.
The game also has powerups which spawn in random locations and can be picked up by moving over them.
Powerups allow for the tank to temporarily gain a new turret which shoots special bullets.

Skills

  • GUI (Graphical User Interface) and 2D graphics
  • Use of third-party libraries (pygame)
  • Game design
  • Image and json file handling
  • Use of multiple source files to organise code

Closer look

I split each component of the game into multiple source files, which makes the code easier to understand and work on.

Levels

level.py handles some of the core aspects of the game, such as controlling the screen and handling rounds, mostly with the Level class.
The Level class handles setting up new rounds, including getting the next level, outputting the scores and resetting the tanks.
All of these are split into their own methods so that the main loop and the general flow of the code is easy to understand.
The Level class controls the levels by importing the settings and images then drawing them to the screen.
The settings are stored in json files making the game more flexible and allowing for custom game modes to be created.
In level.py there is also a class called FPS_Handler which monitors and limits the FPS (frames per second). Although this is a simple class it makes the handling of the FPS much easier to understand.

Sprites

A sprite is a 2D image which appears on the screen and performs a role, for example the sprites in this game are the tanks, bullets and powerups.
sprite.py includes the Sprite class which acts as a base for the other sprites, defining common functions for other classes.
The movement system defined in this class consists of moving forward/backward, rotating and collisions. In the update method the sprite is drawn on the screen.
sprite.py also contains the SpriteHandler class, which stores all of the sprites in the game.
It stores them in groups which makes the handling of sprites more accessable. Additionaly it has functions for adding and removing sprites providing a standard way of using the sprites across the program.

Tanks

The Tank class handles user input through a method called processInput, which chooses a function (eg moving, shooting) to execute depending on the input.
This gives a standard interface to give controls to any tank and allows the controls to be changed in one place.
Bullets of a cetrain tank are handled through only one method named shoot. This method is called the same way even if the tank has a powerup as it decides what type of bullet is shot and handles things such as ammunition.

Powerups

Random powerups spawn in random locations and can be picked up by moving over them.
Powerups allow for the tank to temporarily gain a new turret which shoots special bullets.
Spawning a powerup is done with a simple method createPowerup which handles type, location and timing.
The drone is a power up which spawns a drone that can fly over walls and shoot other tanks. This gives players a unique advantage over other powerups as they cannot escape from the drone which can create interesting gameplay.

Bullets

As there are different types of bullets with different characteristics due to powerups each type of bullet has a different class.
All types of bullets use the same methods but some have differing characteristics. This allows for compatibility when adding new bullet types.
The characteristics of each type of bullet is stored in a json file which is more user friendly than changing it in the code. This allows the configuration to be changed without having to re-build the code.
For powerups that spawn their own bullets (eg drones) I created a class called Bullet_Controller. This handles shooting bullets, shot cooldown and ammunition.

Game

With all of the classes and files discussed above, the game has all of the sections needed.

    The file "Tank Colosseum.py" runs the game and does the following:

  • Imports all sections of the game
  • Initialises each section
  • Contains the main loop which handles events, passing user inputs to the tanks and updating classes
  • Handles exiting the game
