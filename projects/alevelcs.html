Computer Science A Level Project

Description

For my Computer Science A Level I created a website and server where people can upload, play and download games from.
To make the website function I used PHP, with a bit of JavaScript, for dealing with creating the dynamic web pages, for example choosing which game to display and retreiving the data.
The server runs on Linux with Apache and is designed to be managed by an end user so includes programs to help with usability of the server, for example starting and stopping the server.

Skills

  • Back and front end web development
  • Implementing passwords and security over the web
  • Setting up web servers
  • Use of command line interfaces

Closer look

classes.php

classes.php contains the main portion of the code and is included in almost all web pages. This helps keep the code manageable from one main location.
It includes a class called Game and two child classes: DownloadableGame and WebGame.
These contain all the necessary methods for the games such as displaying previews, downloading and running the game on a web page.
There are also FAQ, Report and Search classes alongside some miscellaneous functions which will be discussed later.

Uploading

On the upload page users can fill in a form containing all the information about the game e.g. name, description, genre.
They will also need to upload a zip file containing the game. This will either be a game that will be downloaded and run on the users computer or a game written in javascript that will be able to be run directly on the web.
When it is uploaded the game is stored on the server waiting to be reviewed by the administrators before being allowed to be used by the public.

Administrators

The administrators can log into the site and control the site without having to access the server directly.
They are able to log in by entering the username and password in login.php. This generates a random key of 10 to 25 digits long of random numbers and letters which is used to encrypt the username and password.
A post request to checklogin.php is sent with the key and the encrypted username and password. They are then decrypted with the key and checked.
If they are correct then the key is stored as a cookie and the encrypted username and password is stored as session variables. Each time an admin page is used the username and password is checked on the server.

When they are logged in they can view pending games and search for all games, including pending games.
In manageGame.php the admins can approve games (this moves the game out of the pending folder and makes it available to the public) or deny them.
Admins can test web games directly from the site on gameTest.php.

Downloading and Playing Games

The download page displays information about the game and allows the users to download the game.
On play.php the user can play web games. This page also displays similar games.

Navbar, Footer and Home

Each page has a navigation bar at the top and a footer at the bottom.
The navbar contains all the relevant links to navigate around the site and includes a search bar.
The footer has information about the site and gives a link to the FAQ pages.
On the home page users can view new and popular games.

Search

Users can search for games and filter them by type, genre, player count and rating.
The results can be ordered alphabetically or by upload date, and the order can be reversed too.

FAQ

The admins can add FAQs which can be viewed by users.
The FAQ page shows a preview for all FAQs. Individual FAQs can be searched for and viewed.

Scripts

I wrote a few scripts that assist with running the server:
start.sh starts the server, sets up the log file and runs manageLoginAttempts.sh. This script limits the number of admin login attempts to 5 per hour.
stop.sh stops the server and manageLoginAttempts.sh.
sendEmail.sh sends an email to the administrators, it is used when a game is reported.

Log

A logging system exists where the notable activities can be seen. Each log file is for a particular day.
Errors are logged alongside user activities (rating, reporting and downloading) and admin activities (accepting/denying games).
This can help debugging and track how the site is used.
