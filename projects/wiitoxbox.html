Wii Controller on Xbox

Description

This project allows a Wii remote to be used on an Xbox.
It does this by connecting a Wii remote to a laptop via bluetooth, then the laptop to the Xbox.

Using a Wii remote instead of a normal controller allows for the use of the motion sensors which measures the angle at which the remote is held at.
This provides unique, fun gameplay and can feel more intuitive, for example while driving a car or flying a plane in game.

Skills

  • Interfacing with and emulating input devices
  • Programming in python, including file handling

Closer Look

An Xbox is not able to directly connect to a Wii remote. It could be possible by plugging in some hardware to the Xbox, but I wanted to have a solution only with software.
A computer is able to connect to an Xbox with a remote play feature, but still needs an Xbox controller for input.
As a Wii remote can connect to a computer with bluetooth all that is needed is a program that is able to read the input from the Wii remote and convert it to input from an Xbox controller.

My program does this by using the pygame library to retrieve the input of the buttons and motion sensors, then uses a virtual controller (made with the pyxinput library) to send button presses to the Xbox, through remote play.
A text file is used to store which controls of the Wii remote translate to which output of the Xbox controller.
The controls file can contain two different control schemes, the active scheme is changed by pressing the home button. This can be used for having a different controls for navigating menus, or for different modes during gameplay.
Each line of the file holds one button mapping in this syntax: wiiButton:xboxButton. This makes changing and creating controls very easy to understand.

Two additional programs were created for this project:

  • wmTest.py would output every button press and the state of each axis (nunchuck and rotation), it was used to find out the values of each button and to make sure every button is working properly.
  • wmCaliration.py is used to get the minimum, maximum and rest values for the nunchuck and motion sensors. The values from this program are used in the main program.
